## 风险评估

- 聊天主链路在离线回退模式下与真实 Provider 模式行为差异
- Tool 参数校验前后端一致性可能受字段命名变化影响
- 本地 Windows 环境下代理/网络差异可能影响 GitHub、Provider 或 MCP 可用性
- Docker 首次构建耗时较长，若本地网络波动可能导致镜像拉取失败

## 回滚方案

1. 回退到上一稳定 tag/commit（建议 `v0.0.x` 稳定点）
2. 重新构建并启动：
   - `docker compose down`
   - `docker compose up --build -d app qdrant`
3. 验证：
   - `curl.exe -sS http://127.0.0.1:8001/api/health`
   - 打开 `http://localhost:1420` 并完成一轮会话发送
4. 若仍异常，恢复上一稳定产物并暂停本次发布
