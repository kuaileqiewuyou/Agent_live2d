## 风险评估

- 聊天主链路在离线回退模式与真实 Provider 模式下仍可能存在行为差异
- Tool 参数校验前后端一致性仍需持续关注字段命名演进
- 本地 Windows 网络/代理环境差异可能影响 GitHub、Provider、MCP 可用性
- Docker 首次构建耗时较长，网络波动时可能拉镜像失败

## 回滚方案

1. 回滚到上一个稳定 tag/commit（建议 `v0.0.x` 稳定点）
2. 重新构建并启动：
   - `docker compose down`
   - `docker compose up --build -d app qdrant`
3. 验证：
   - `curl.exe -sS http://127.0.0.1:8001/api/health`
   - 打开 `http://localhost:1420` 并完成一轮会话发送
4. 若仍异常，恢复上一个稳定产物并暂停本次发布
