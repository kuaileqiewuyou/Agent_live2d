# v0.1.0-rc1 手动验收清单（Desktop / Web）

## 使用说明

- 执行人：发布责任人或指定验收人
- 执行环境：Windows 本地开发机（含 Docker Desktop、Node、Rust/Tauri）
- 结果标记：`[x]` 通过，`[ ]` 未通过，附简短备注

---

## A. 预检查

- [ ] `npm install` 成功，无依赖安装错误
- [ ] `npm run desktop:doctor` 通过（cargo/rustc/tauri cli 全部 OK）
- [ ] Docker Desktop 已启动且可用（`docker version` 正常）

---

## B. Docker 后端链路验收

### B1. 启动与健康

- [ ] 执行 `npm run local:up` 成功
- [ ] 访问 `http://127.0.0.1:8001/api/health` 返回 `success=true`
- [ ] `docker compose ps` 显示 `agent-live2d-backend` 为 `healthy`
- [ ] `docker inspect agent-live2d-qdrant --format "{{.State.Health.Status}}"` 返回 `healthy`

### B2. 关键功能烟测

- [ ] 打开 Web 页面后可正常创建会话
- [ ] 聊天发送一条普通消息可返回 assistant 回复
- [ ] Tool Panel 可选中 Skill/MCP 并发送
- [ ] “只选工具不输入消息”可触发兜底指令并成功返回

---

## C. Desktop 链路验收（Tauri）

### C1. 启动

- [ ] 执行 `npm run local:desktop` 成功拉起（不出现挂载失败）
- [ ] 不出现 `os error 5`（拒绝访问）导致的 `tauri:dev` 启动失败
- [ ] 桌面窗口成功打开并显示聊天主界面

### C2. 交互

- [ ] 切换会话 A/B 后，输入草稿可隔离与恢复
- [ ] 调整背景透明度可保存成功，无“无法连接后端服务”报错
- [ ] Tool Panel 的 typed params：
  - [ ] 合法参数可发送并执行
  - [ ] 非法参数被拦截并显示错误提示

### C3. 稳定性

- [ ] 关闭 Desktop 后再次 `npm run local:desktop` 可再次启动
- [ ] 启动后端容器状态保持正常（app/qdrant 均 healthy）

---

## D. Web 链路验收（可选但建议）

- [ ] 执行 `npm run local:web` 成功
- [ ] 浏览器端聊天与 Tool Panel 行为与 Desktop 一致
- [ ] 页面刷新后背景设置可恢复
- [ ] 切换会话后可恢复该会话最近工具选择

---

## E. 收尾

- [ ] 执行 `npm run local:down` 成功，容器全部停止
- [ ] 验收结果回填 release ticket（通过项/失败项/备注）
- [ ] 如有失败项，附复现步骤与日志片段
