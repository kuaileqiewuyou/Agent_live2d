# MD 对齐与接手文档治理设计

**日期**: 2026-04-18
**状态**: Draft
**范围**: `AGENTS.md` / `README.md` / `CONTRACT.md` / `TASK.md`

---

## 1. 背景与目标

当前仓库文档信息存在跨文件重复定义与边界不清问题，导致接手、联调和迭代时出现“同一信息多处漂移”。
本设计目标是在不改动代码与配置的前提下，完成四份核心 Markdown 的结构化重排与职责对齐。

本次目标：
- 明确四份文档的唯一职责边界
- 建立“单一真源 + 引用链接”机制，避免重复定义
- 让新接手成员能够通过 README 快速进入，再按 AGENTS/CONTRACT/TASK 深入

---

## 2. 非目标

本次不包含以下内容：
- 不修改任何后端/前端代码
- 不修改 CI、数据库、环境配置
- 不新增文档体系外的流程工具或自动化脚本
- 不扩展到 release 文档与历史计划文档的重写

---

## 3. 文档职责边界（单一真源）

### 3.1 `AGENTS.md`（协作规则唯一真源）

负责：
- 项目阶段目标与约束
- 分层架构原则与禁止事项
- superpower-plus 强制流程
- 关键操作确认机制
- 实施前必报项

不负责：
- 具体 API 字段或返回样例细节
- 当前里程碑完成度追踪

### 3.2 `CONTRACT.md`（接口契约唯一真源）

负责：
- API 路径、请求/响应字段、错误码
- SSE 事件与幂等等可测试行为
- Ops / MCP / Memory 等能力契约

不负责：
- 协作流程约束
- 运行指引与入门说明

### 3.3 `README.md`（接手与运行唯一入口）

负责：
- 项目简介与能力边界
- 本地启动、Docker、联调路径
- 架构总览与文档导航

不负责：
- 详细接口契约定义
- 里程碑状态跟踪

### 3.4 `TASK.md`（执行状态唯一真源）

负责：
- 当前基线与里程碑
- 优先级、验收标准、风险与执行顺序

不负责：
- 接口字段定义
- 协作流程规则定义

---

## 4. 四份文档重排结构

### 4.1 `AGENTS.md` 建议结构

1. 项目定位与阶段目标
2. 架构原则与目录约定
3. superpower-plus 强制流程
4. 关键操作先确认清单
5. 实施前必报三项
6. 决策原则与禁止事项
7. 文档优先级规则（引用 `CONTRACT.md`）

### 4.2 `CONTRACT.md` 建议结构

1. 通用响应与命名约定
2. 资源分组接口契约
3. SSE 事件契约
4. 错误契约
5. 幂等与并发约定
6. Ops 扩展契约

### 4.3 `README.md` 建议结构

1. 项目简介（当前能力边界）
2. 快速开始（本地 / Docker / Web / Tauri）
3. 最小联调路径
4. 架构总览（目录 + 职责）
5. 文档导航（AGENTS / CONTRACT / TASK）
6. FAQ（编码、端口、依赖）

### 4.4 `TASK.md` 建议结构

1. 当前基线
2. 里程碑列表（进行中/未开始/已完成）
3. 每个里程碑验收标准
4. 风险与缓解
5. 建议执行顺序
6. DoD

---

## 5. 同步机制（防漂移）

### 5.1 单一真源规则

- 协作规则只在 `AGENTS.md`
- 接口契约只在 `CONTRACT.md`
- 接手与运行只在 `README.md`
- 执行状态只在 `TASK.md`

### 5.2 联动更新规则

- 接口行为变更：必须同 PR 更新 `CONTRACT.md`
- 流程约束变更：只更新 `AGENTS.md`
- 阶段状态变更：只更新 `TASK.md`
- `README.md` 只维护入口与导航，不复制契约细节

### 5.3 评审检查项

- 是否出现“同一定义多文件重复”
- 是否把契约细节写进了 README
- 是否把流程规则写进了 TASK
- 是否存在无引用来源的断言

---

## 6. 实施方案

### 6.1 改动范围

仅改：
- `AGENTS.md`
- `README.md`
- `CONTRACT.md`
- `TASK.md`

不改：
- 代码、测试、CI、数据库与环境配置文件

### 6.2 实施顺序

1. `AGENTS.md` 先定规则边界
2. `CONTRACT.md` 对齐接口真值
3. `README.md` 重排接手入口与导航
4. `TASK.md` 对齐里程碑与验收
5. 交叉检查四份文档一致性

### 6.3 验证方式

- 文本层检查：章节边界是否清晰、是否重复定义
- 路径检查：README 导航路径有效
- 一致性检查：TASK 中提及能力与 CONTRACT 能力一致

---

## 7. 风险与处理

- 风险：重排时误删有效信息
  - 处理：以“迁移而非删除”为原则，先保留再收敛
- 风险：README 再次膨胀为契约文档
  - 处理：README 仅保留入口级内容，细节一律链接 CONTRACT
- 风险：后续迭代不遵守单一真源
  - 处理：在 AGENTS 增加文档变更优先级和联动规则

---

## 8. 完成定义

本次 MD 对齐完成需同时满足：
- 四份文档完成结构化重排
- 四份文档职责边界明确且无冲突
- 关键信息只在唯一真源定义，其他文档改为引用
- 不引入任何代码或配置改动
